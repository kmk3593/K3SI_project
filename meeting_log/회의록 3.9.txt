챗봇시스템 시나리오
  - 사용자시점
1. 정상적인 사용시
 사용자는 채팅을 입력한다.
	~~~가 궁금해
 챗봇은 해당 문장을 메인서버에 전송
 메인서버는 해당문장을 분석
 알맞는 답변을 DB에서 확인
 DB는 해당 답변을 전송
 메인서버는 수신한 답변을 클라이언트로 전송
 챗봇은 수신한 답변을 출력
 사용자는 정보를 확인

2. 정보갱신요구
 사용자는 채팅을 입력한다.
	~~~가 궁금해
 챗봇은 해당 문장을 메인서버에 전송
 메인서버는 해당문장을 분석
 알맞는 답변을 DB에서 확인
 DB는 해당 답변을 전송
 메인서버는 수신한 답변을 클라이언트로 전송
 챗봇은 수신한 답변을 출력
 사용자는 정보를 확인
 사용자가 정보가 너무 과거의 것이라 판단
 챗봇의 채팅하단의 갱신요구버튼을 클릭
 해당 갱신요구가 메인서버로 전달
 메인서버에서 DB로 해당 항목에 대한 갱신요구 전송
 DB는 해당 항목에 대하여 갱신필요 항목 활성화
 관리자는 갱신요구테이블을 관리자 사이트에서 확인

3. 리포트 기능
 사용자는 채팅을 입력한다.
	~~~가 궁금해
 챗봇은 해당 문장을 메인서버에 전송
 메인서버는 해당문장을 분석
 알맞는 답변을 DB에서 확인
 DB는 해당 답변을 전송
 메인서버는 수신한 답변을 클라이언트로 전송
 챗봇은 수신한 답변을 출력
 사용자는 정보를 확인
 
 오류발생한 상황을 인지
 우측 상단의 설정에서 오류보고항목을 선택
 발생 시각과 발생정황을 입력
 클라이언트는 메인서버로 해당 데이터를 전송
 메인서버는 해당 데이터를 수신
 해당 데이터를 DB로 전송
 DB는 해당 데이터를 테이블에 저장
 관리자가 리포트 열람시 DB는 해당데이터를 메인서버로 전송
 메인서버는 데이터를 관리자 사이트로 전송
 관리자가 리포트를 열람후 확인완료한 경우 해당 데이터 열람, 해결 여부 기록

2. 오류보고 확인
 관리자 사이트에 접속하여 로그인을한다.
 오류보고에 접근한다.

 메인서버는 DB에 오류리포트항목을 요청한다.

 메인서버는 수신한 오류리포트항목을 클라이언트에 전송한다.
 클라이언트는 오류리포트항목을 출력한다.
 
3. 데이터 추가
 관리자 사이트에 접속하여 로그인을한다.
 데이터 추가에 접근한다.
 추가할 데이터가 담긴 엑셀파일을 클라이언트를 통해서 메인서버에 전송한다. 
 메인서버는 수신받은 파일을 자체 코드를 통해서 DB로 전송한다.
